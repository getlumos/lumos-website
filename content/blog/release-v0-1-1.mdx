# LUMOS v0.1.1 Released

We're excited to announce LUMOS v0.1.1, featuring important security enhancements and developer experience improvements.

## What's New

### üîí Security Enhancements

#### Path Traversal Protection

The CLI now validates all file paths to prevent path traversal attacks:

```bash
# ‚ùå Blocked
lumos generate schema.lumos --output ../../etc/passwd

# ‚úÖ Allowed
lumos generate schema.lumos --output ./src/generated/
```

#### User-Defined Type Validation

Early detection of undefined types during transformation:

```rust
// ‚ùå Error: undefined type 'InvalidType'
struct Player {
    inventory: [InvalidType],
}

// ‚úÖ Clear error message with location
// Error: Undefined type 'InvalidType' in field 'Player.inventory'
```

### ‚ö†Ô∏è TypeScript Precision Warnings

Automatic JSDoc warnings for u64/i64 fields:

```typescript
// Generated TypeScript
export interface Account {
  /**
   * LUMOS Warning: This field is u64 in Rust but 'number' in TypeScript.
   * JavaScript numbers are safe up to 2^53-1 (9,007,199,254,740,991).
   * For larger values (common with Solana lamports), consider using bigint.
   */
  balance: number;
}
```

### üìù Improved Error Messages

Better error reporting with source location tracking:

```
Error: Invalid type 'u256' at line 15, column 12
  |
15|     value: u256,
  |            ^^^^ unsupported type
  |
Help: Use u64, u128, or [u8; 32] instead
```

### üêõ Bug Fixes

- Fixed enum discriminant generation for TypeScript
- Resolved issue with nested Option types
- Corrected Borsh schema generation for arrays
- Fixed LSP hover tooltips for Solana types

## Upgrade Guide

### Update CLI

```bash
cargo install lumos-cli --version 0.1.1
```

### Verify Installation

```bash
lumos --version
# lumos-cli 0.1.1
```

### Breaking Changes

**None!** This is a backwards-compatible release.

### Recommended Actions

1. **Review TypeScript number usage**: Check u64 fields that might exceed JavaScript safe integers
2. **Update error handling**: Take advantage of improved error messages for debugging
3. **Run security audit**: Verify your build scripts don't use path traversal

## Migration Notes

No migration needed - existing schemas work without changes.
